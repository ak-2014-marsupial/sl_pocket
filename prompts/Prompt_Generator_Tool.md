# ІНСТРУКЦІЯ ДЛЯ ОНОВЛЕННЯ НАВЧАЛЬНИХ ПРОМПТІВ (Meta-Prompt)

Цей файл призначений для Gemini CLI або іншої LLM для інкрементального оновлення бази знань.

## ЗАВДАННЯ:
Оновлювати файли `Prompt_PartX.txt` новими матеріалами з `documents/` порційно, зберігаючи структуру та чистоту даних.

## АЛГОРИТМ РОБОТИ:
1. **Аналіз документів:** Вивести список усіх `.md` файлів у папці `documents/`.
2. **Запит користувача:** Запитати: "Які з цих уроків додати або оновити? (вкажіть номери через кому або діапазон)".
3. **Визначення цільового файлу:**
   - Уроки 01-10 -> `Prompt_Part1.txt`
   - Уроки 11-20 -> `Prompt_Part2.txt`
   - Уроки 21-30 -> `Prompt_Part3.txt`
   - Уроки 31-40 -> `Prompt_Part4.txt`
   - Уроки 41-50 -> `Prompt_Part5.txt`
4. **Збір даних для вибраних уроків:**
   - **Vocab (100%):** Всі пари "слово — переклад" з таблиць.
   - **Grammar Essentials:** Формули та приклади.
   - **Practice & Translation:** УСІ речення з вправ та блоків перекладу.
5. **Оновлення файлу:**
   - Перевірити наявність файлу. Якщо його немає — створити.
   - **Обов'язковий заголовок:** Переконатися, що на початку файлу є інструкція для вчителя (див. нижче). Якщо немає — додати.
   - **Заміна (Clean-up):** Якщо блок цього уроку вже є у файлі — ПОВНІСТЮ замінити його новим вмістом. Якщо немає — додати в кінець файлу.

## СТРУКТУРА ЗАГОЛОВКА ФАЙЛІВ-ПРОМПТІВ:
Кожен текстовий файл ПОВИНЕН починатися з цих рядків:
- "Твоя мета — переконатися, що учень засвоїв КОЖНЕ слово та КОЖНУ фразу з обраної теми."
- "Проводь тестування, поки не пройдешся по всьому списку словника та прикладів."
- "Постійно вимагай використання Polite English softeners."

## ЯК ЗАПУСТИТИ:
Напиши мені: "Онови промпти згідно з новою інструкцією в Prompt_Generator_Tool.md".
